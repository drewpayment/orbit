# Configuration

Orbit is configured through environment variables. The primary configuration lives in `orbit-www/.env`.

## Core Settings

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URI` | Yes | MongoDB connection string |
| `PAYLOAD_SECRET` | Yes | Secret for Payload CMS (random 32+ chars) |
| `NEXT_PUBLIC_APP_URL` | Yes | Public URL of your Orbit instance |
| `ENCRYPTION_KEY` | Yes (prod) | AES-256 encryption key (32 bytes, base64-encoded) for secret management |
| `ORBIT_INTERNAL_API_KEY` | Yes | Shared key for inter-service authentication (must match temporal-worker) |
| `BETTER_AUTH_SECRET` | Yes (prod) | Secret for Better Auth sessions |

## Documentation Site

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_DOCS_URL` | No | URL of your orbit-docs instance. When set, shows "Docs" links in the header and sidebar. Leave unset to hide. |

See [Docker Compose — Documentation Site](/docs/self-hosting/docker-compose#documentation-site-optional) for setup details.

## GitHub App Integration

For repository management, template instantiation, container builds, and `.orbit.yaml` sync:

| Variable | Description |
|----------|-------------|
| `GITHUB_APP_ID` | Your GitHub App's ID |
| `GITHUB_APP_CLIENT_ID` | OAuth client ID |
| `GITHUB_APP_CLIENT_SECRET` | OAuth client secret |
| `GITHUB_APP_WEBHOOK_SECRET` | Webhook signature secret |
| `GITHUB_APP_PRIVATE_KEY_BASE64` | Base64-encoded private key |
| `NEXT_PUBLIC_GITHUB_APP_NAME` | GitHub App slug (e.g., `orbit-idp`) — **build-time variable** |

See the [GitHub App Setup](/docs/getting-started/github-app) guide for complete step-by-step instructions on creating and configuring the app.

## Email (Optional)

For email notifications (signup verification, password resets):

| Variable | Description |
|----------|-------------|
| `RESEND_API_KEY` | [Resend](https://resend.com) API key |
| `RESEND_FROM_EMAIL` | Sender email address |

## Kafka Setup

After starting the dev environment, register your local Redpanda cluster:

1. Go to **Admin Panel → Kafka Providers** and create a provider:
   - Name: `redpanda`
   - Adapter Type: `apache`
2. Go to **Kafka Clusters** and create a cluster:
   - Provider: select your Redpanda provider
   - Connection Config: `{"bootstrap.servers": "localhost:19092"}`
3. Go to **Kafka Environment Mappings** and create a mapping:
   - Environment: `development`
   - Cluster: select your cluster
   - Is Default: `true`

See the [Kafka Self-Service](/docs/features/kafka) docs for more details.
