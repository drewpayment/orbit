# Docker Compose Deployment

Docker Compose is the simplest way to get Orbit running. It's ideal for:

- Local development
- Evaluation and demos
- Small team deployments (< 50 users)

## Quick Start

```bash
git clone git@github.com:drewpayment/orbit.git
cd orbit

# Copy environment config
cp orbit-www/.env.example orbit-www/.env
# Edit orbit-www/.env with your settings (see Configuration)

# Start everything
make dev
```

## What Gets Started

The Docker Compose setup launches:

| Service | Image | Purpose |
|---------|-------|---------|
| `orbit-www` | Built from source | Frontend + CMS |
| `mongodb` | `mongo:7` | Application database |
| `postgres` | `postgres:15` | Temporal persistence |
| `redis` | `redis:7` | Caching & sessions |
| `temporal` | `temporalio/auto-setup` | Workflow engine |
| `temporal-ui` | `temporalio/ui` | Workflow monitoring |
| `temporal-worker` | Built from source | Workflow execution |
| `meilisearch` | `getmeili/meilisearch` | Full-text search |
| `minio` | `minio/minio` | Object storage |
| `redpanda` | `redpandadata/redpanda` | Kafka-compatible broker |
| `redpanda-console` | `redpandadata/console` | Kafka topic UI |

## Production Considerations

Docker Compose works for small deployments, but for production you should:

- Use **external managed databases** (MongoDB Atlas, Amazon RDS) instead of containerized DBs
- Set up **backup and restore** procedures
- Configure **TLS termination** via a reverse proxy (nginx, Caddy, Traefik)
- Set up **monitoring** (Prometheus + Grafana or similar)
- Consider [Kubernetes](/docs/self-hosting/kubernetes) for larger deployments

## Reverse Proxy Example

Here's a minimal Caddy config to put Orbit behind TLS:

```
orbit.example.com {
    reverse_proxy localhost:3000
}

temporal.example.com {
    reverse_proxy localhost:8080
}
```
